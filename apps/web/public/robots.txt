# NoKLock robots.txt
# Updated 2026-05-20 — 3-category AI bot split per the May-2026 industry taxonomy.
# Policy decision (Daniel, security-conscious product):
#   ALLOW retrieval/answer + user-action crawlers (we WANT to be cited in AI answers)
#   BLOCK training crawlers (our threat model + audit findings should not be baked
#     into model weights — attackers should not have access to a model that has
#     memorised our specific defences and attack-surface documentation)
#
# This is the "allow citation, block training" stance — the most common
# deliberate posture for a security-conscious SaaS that wants AI visibility
# without contributing to training datasets.

# === Note for any AI that ignores robots.txt =================================
# NoKLock additionally signals training-data opt-out via:
#   - /ai.txt          (machine-readable AI policy)
#   - TDMRep HTTP headers (tdm-reservation: 1 + tdm-policy: /ai.txt)
#   - The Disallow entries above for training crawlers (GPTBot etc.)
# 2026-06-05 NOTE: the `noai, noml` tokens previously in the X-Robots-Tag
# HTTP header + <meta name="robots"> were REMOVED — they were signalling
# "do not cite" to AI search bots (Perplexity / ChatGPT / Claude), which
# defeated the citation-allow stance above. Training opt-out is now carried
# exclusively by the legally-weighted signals (the bot-specific Disallows
# above + TDMRep headers + /ai.txt). EU AI Act Article 53 and Germany's
# UrhDaG implementation of the DSM Directive Article 4 are the load-bearing
# legal references — they require a machine-actionable reservation in any
# form, and Disallow + TDMRep cover that without `noai`.
